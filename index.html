Units of Digital Storage

Understanding bits, bytes, and the SI and IEC prefix families

📐 The Fundamental Units

All digital data is stored as binary digits. The two foundational units are:

Bit (b) — the smallest unit of data. A single binary digit: either 0 or 1. A bit represents one of two states (off/on, false/true).

Byte (B) — 8 bits grouped together. One byte can represent 2⁸ = 256 different values (0–255). A single character in basic ASCII encoding takes 1 byte.

1 Byte = 8 bits  |  1 Nibble = 4 bits = ½ Byte

⚠️ Two Prefix Systems — SI vs IEC

There are TWO systems of prefixes used with storage units. This causes confusion — you must know both!

SI (decimal) prefixes — based on powers of 10. Used in marketing (hard drive manufacturers use these — that's why drives appear smaller than advertised!).

IEC (binary) prefixes — based on powers of 2. Used in operating systems and computing. Identified by the extra "i" (e.g. Kibibyte, Mebibyte).

SI: 1 KB = 1 000 bytes (10³)    1 MB = 1 000 000 bytes (10⁶) IEC: 1 KiB = 1 024 bytes (2¹⁰)   1 MiB = 1 048 576 bytes (2²⁰)

💡 Memory Trick

  • SI (decimal, powers of 10): Kilo, Mega, Giga, Tera — "King Midas Got Treasure"
  • IEC (binary, powers of 2): Kibi, Mibi, Gibi, Tibi — same words but with the "i" to show binary
  • For IGCSE, you often see KB used to mean 1024 bytes (the old informal use). Always check the context!
Binary steps: × 1024 each time  (2¹⁰ = 1024) Decimal steps: × 1000 each time  (10³ = 1000)

Converting Between Units

Multiplying and dividing through the binary and decimal prefix ladders

📐 The Conversion Rules

In the IEC binary system each step up is ×1024 and each step down is ÷1024:

bits ×8 ÷ bytes ÷ 1024→ KiB ÷ 1024→ MiB ÷ 1024→ GiB ÷ 1024→ TiB TiB × 1024→ GiB × 1024→ MiB × 1024→ KiB × 1024→ bytes × 8→ bits

In the SI decimal system each step is ×1000:

bytes ×1000→ KB ×1000→ MB ×1000→ GB ×1000→ TB
Worked Example 1: Convert 5 MiB to bytes Step 1: 5 MiB × 1024 = 5120 KiB Step 2: 5120 KiB × 1024 = 5 242 880 bytes Answer: 5 MiB = 5 242 880 bytes Shortcut: 5 × 2²⁰ = 5 × 1 048 576 = 5 242 880 bytes ✓
Worked Example 2: Convert 12 288 bytes to KiB 12 288 ÷ 1024 = 12 KiB Answer: 12 288 bytes = 12 KiB
Worked Example 3: Convert 2.5 GiB to MiB 2.5 × 1024 = 2560 MiB Answer: 2.5 GiB = 2560 MiB

Text Representation

How characters are encoded as binary numbers using ASCII and Unicode

📝 ASCII — American Standard Code for Information Interchange

ASCII is the original character encoding standard. It assigns a unique number (code point) to each character.

  • Standard ASCII uses 7 bits → 2⁷ = 128 characters (0–127)
  • Extended ASCII uses 8 bits → 2⁸ = 256 characters (0–255)
  • Covers: uppercase A–Z, lowercase a–z, digits 0–9, punctuation, control characters
  • Each character = 1 byte of storage
File size (bytes) = Number of characters × 1 byte

🌐 Unicode — Universal Character Encoding

ASCII only covers English and basic symbols. Unicode was created to represent every character from every language on Earth.

  • UTF-8 — variable width: 1 to 4 bytes per character. Backward compatible with ASCII.
  • UTF-16 — uses 16 bits (2 bytes) per character for the most common characters (Basic Multilingual Plane). Can represent 2¹⁶ = 65 536 characters.
  • UTF-32 — uses 32 bits (4 bytes) per character. Fixed width, covers all Unicode.
UTF-16 file size (bytes) = Number of characters × 2 UTF-32 file size (bytes) = Number of characters × 4
Worked Example — Text File Size (UTF-16): "To be or not to be, that is the question." Count characters (including spaces and punctuation): 42 characters UTF-16: 42 × 2 bytes = 84 bytes File size = 84 bytes

Image Representation

Pixels, colour depth, resolution, and file size calculations

🖼️ How Images are Stored — Bitmaps

A bitmap (raster) image stores colour data for every individual pixel. Each pixel is one coloured dot on the screen.

  • Resolution — the dimensions of the image in pixels, e.g. 1920 × 1080
  • Colour depth — the number of bits used to store the colour of each pixel
  • More bits per pixel = more possible colours
Number of colours = 2^(colour depth in bits) File size (bits) = Width × Height × Colour depth (bits per pixel) File size (bytes) = Width × Height × Colour depth ÷ 8 File size (KiB) = Width × Height × Colour depth ÷ 8 ÷ 1024

Let original size = S KiB. New size = S × (2/8) = S/4 KiB Reduction: S − S/4 = 3S/4 = 18 KiB S = 18 × 4/3 = 24 KiB Original file = 24 KiB ✓

Task 2 — Colour Reduction

Exam question: After converting a 256-colour bitmap to a 4-colour format, its size decreased by 18 KiB. What was the original file size in KiB?

Hint: Work out the ratio of bits per pixel before and after. Let original = S, write an equation for the reduction.
256 colours = 8 bits/px. 4 colours = 2 bits/px. New size = original × (2/8) = original/4. Reduction = 3/4 of original = 18 KiB. Original = 18 × 4/3 = 24 KiB

Sound Representation

Sampling, sample rate, bit depth, channels, and audio file size

🎵 How Sound is Digitised — ADC Process

Sound is an analogue (continuous) wave. To store it digitally, an Analogue-to-Digital Converter (ADC) takes regular snapshots called samples.

  • Sample rate (sampling frequency) — how many samples are taken per second, measured in Hertz (Hz) or kHz. More samples → higher quality but larger file.
  • Bit depth (sample resolution) — how many bits are used to record each sample's amplitude. More bits → more precise amplitude → better quality.
  • Channels — mono (1 channel), stereo (2 channels), quad (4 channels), etc.
File size (bits) = Sample rate × Bit depth × Channels × Duration (seconds) File size (bytes) = Sample rate × Bit depth × Channels × Duration ÷ 8

📊 Common Audio Standards

The Nyquist theorem states the sample rate must be at least twice the highest frequency you want to capture. Human hearing tops at ~20 kHz, hence CD uses 44.1 kHz.

Worked Example 1 — 4-channel recording: 4-channel (quad), 32 kHz, 32-bit, 2 minutes. Find file size in MiB. Duration: 2 min = 120 seconds File size (bits) = 32 000 × 32 × 4 × 120 = 491 520 000 bits ÷ 8 = 61 440 000 bytes ÷ 1 048 576 ≈ 58.6 MiB Nearest integer multiple of 10 → 60 MiB ✓
Worked Example 2 — Transmission time comparison: File 1: sent in 30s. File 2: 2× bit depth, 1.5× sample rate, channel 4× faster. Time for File 2? File 2 size relative to File 1: × 2 (bit depth) × 1.5 (sample rate) = × 3 Channel to City B is 4× faster than City A Time = 30 × 3 ÷ 4 = 22.5 → wait, let's recalculate: File 1 size = rate × 30. File 2 = 3 × File 1 size. Channel B speed = 4 × Channel A speed. Time B = File 2 size ÷ Channel B speed = (3 × File 1) ÷ (4 × rate) = 3/4 × 30 = 22.5s But sample rate DECREASED by 1.5×, so File 2 = 2 × (1/1.5) = 2/1.5 × File 1 = 4/3 × File 1 Time B = (4/3 × 30) ÷ 4 = 40/4 = 10 seconds Answer: 10 seconds ✓

Task 3 — Transmission Time Comparison

Exam question: A music track was digitised and sent to City A in 30 seconds. The same track was re-digitised with 2× the bit depth and 1.5× lower sample rate (i.e., sample rate divided by 1.5). No compression. The channel to City B is 4× faster than City A's channel. How many seconds did the transfer to City B take?

Hint: First find the new file size as a multiple of the original. Then apply the new channel speed.
Size factor = 2 × (1/1.5) = 2/1.5 = 4/3. New file is 4/3 × original. Transfer time = 30 × (4/3) ÷ 4 = 30 × 4/(3×4) = 30/3 = 10 seconds ✓
